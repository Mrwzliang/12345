sublime_text3_emmet

字数统计: 725字   |   阅读时长: 2分
  • emmet

1.连续搞了13小时终于装好了emmet，python果然也不好惹，自己到底还是个弟中弟。
2.本文基本都是手动安装插件，因为基本不可以自动安装，都会有问题。
ok
——————————————————————————————————————————————————————————————————————————————————————————————————————————————————————————————————
1、安装package control。
首先检查一下你的sublime有没有package control，在Preferences中如果有package settings和package control，说明有package control，如果没有，可以自动安装，但是可能不会成功，所以建议手动安装，下载安装包，然后放入Preferences->Browses packages->打开文件的上一级->Installed Packages中，重启sublime软件，按照上面的方法检查判断是否成功安装。

2、安装emmet插件。
Preferences->package control->搜索emmet安装，但是同样，自动可能不会成功，而且在这个过程中又出现了新的问题（there are no packages available….
这是因为在sublime进行插件下载时，会调用channel_v3.json文件，点击Preferences->package settings->Package Control ->Setting Default，可以看到该文件是放置在网络中进行读取的，而由于GFW的原因，导致无法读取该文件，这也就是导致插件无法下载的原因。
解决方法是：1、下载channel_v3.json文件。
2、接着在Preferences->package setting->package control ->setting user设置中，添加代码（添加channel_v3.json文件所在位置的代码），至此，就可以正常使用install package下载插件。我添加的代码如下：

“channels”:

[
“F:/sublime/channel_v3.json”

]
然后用上面的方法安装，之后重启sublime软件。
3、安装PyV8。
安装成功emmet插件之后，sublime软件会自动下载PyV8，没错，又是好久的等待，然后失败，只能手动下载。在这里提醒大家一定要清楚版本，下载相应的安装包，网址：https://github.com/emmetio/pyv8-binaries#readme
将压缩包放到Preferences->Browses packages->打开文件的上一级->创建PyV8文件夹->黏贴刚才下载的安装包并解压，重启sublime软件，大功告成。
最后，你会发现按tab依然不能补全，这是因为Emmet默认的快捷键是Ctrl+E，我们需要将其设置成常用的Tab键。
在Preferences–>PackageSettings–>Emmet–>KeyBindings–>User，将以下信息粘贴进去即可。
[{“keys”: [“tab”], “args”: {“action”: “expand_abbreviation”}, “command”: “run_emmet_action”, “context”: [{“key”: “emmet_action_enabled.expand_abbreviation”}]}]
重启Sublime软件，此时我们便可以进行代码补全操作了。

可能还会出现的问题：
1.打开控制台会出现：PyV8.JSContext() AttributeError: ‘module’ object has no attribute ‘JSContext’这个是你pyV8的安装包没有放到正确位置，不应该放到Preference－－Browser Packages的文件夹，而应该放到上层文件夹的Installed Packages这个文件夹。即本着PyV8这个文件夹必须和Emmet同在一个父文件夹下为原则。
2.解压出来后，以前的的安装包删除了，不要放在运行目录里。
