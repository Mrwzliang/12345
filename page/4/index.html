深入浅出MVC

发表于 2018-01-14 分类于 java

标题一：早期开发模型介绍

model1时我们根本就没有controller,主要就是浏览器，jsp，javabean（包含service,dao）,在项目都比较大，业务逻辑比较复杂的情况下，肯定行不通，jsp等于干了视图与控制器的事，逻辑代码写在jsp里复用性差，维护性差。model2模型对model1进行了改良，明确了分工。由浏览器，servlet,javabean(业务逻辑部分）jsp组成。添加了servlet组件即作为控制器，去处理请求（比如是否有权限继续，携带的参数是否正确），
接下来再去调用业务逻辑，再返回结果交给servlet处理，再找到对应的jsp给浏览器展示数据。
mvc模型也存在弊端，其一就是展现数据比较慢（jsp需要翻译成java,编译成class，然后展示），还有就是异步交互并不方便（响应回ajax引擎之后的数据处理，需要有dom编程的功底），以及前端的逻辑代码与数据的展现部分代码过于耦合（目前vue框架已经实现分离）。所以mvvc即解决这种弊端的一种模型。

mybatis应用小结

发表于 2018-01-11 分类于 java

1.mybatis(前身是apache的ibatis)的出现解决了JDBC的若干缺陷，而且直接基于JDBC做了简单的映射包装，所以从性能上来说JDBC>Mybatis>Hibernate.

2.Hibernate与Mybatis都是ORM(对象关系映射)持久化框架，但是Mybatis是将接口与POJO对象映射成数据库中的记录，Hibernate仅仅是通过pojo对象
Mybatis除了支持简单sql查询，也支持高级映射(关联查询)与存储过程，实现了自动对sql参数的设置，自动对结果集进行解析与封装。

3.虽然mybatis支持注解与xml方式配置，但是一般来说我们采用xml配置方式，这样能将java代码与sql语句很好的隔离下来。

4.但是正因为她只是做了简单的封装，所以需要手动写sql(通用mapper只是简单查询)，所以如果对性能要求较高的电商项目会用mybatis，而对于业务逻辑复杂，不太在乎执行效率的传统行业，一般会使用Hibernate.

5.mybatis两种配置文件，一个全局的，还有一个映射文件，通过这两个配置文件就可以构建sqlsessionFactory相当于hibernate里的sessionFactory,打开sqlsession的会话。在通过session实现CRUD,接下来就是Executor(执行器)，是一个接口，有两种实现，一个是缓存的实现，还有一个是数据库的实现。最底层的Mapper Statement主要是在三个方面，即sql语句，参数类型(hashmap,基本数据类型，包装类型pojo)，结果集类型。

6.mapper接口的动态代理实现，主要是两点，其一就是映射文件中的命名空间与Mapper接口的全路径一致。其二映射文件中的statementId与Mapper接口的方法名保持一致。

7.mybatis-config.xml的配置，有严格的顺序之分。

关于反射

1.首先，在java中，java程序如何运行的？很明显是将.java文件编译成.class文件，即字节码文件，是可以被jvm直接操作的，jvm通过类加载器ClassLoader把class文件加载到内存中，这样就可以使用这个类中的成员变量与方法了，同时被加载到内存中class文件就会变成一个Class类的对象。而我们的反射需要依赖Class对象。任何一个class文件都是Class这个类的一个实例对象。Class对象通过(Field getField(String name),Method getMethod(),Constructor getConstructor())。

2.所以反射就是通过一个类的Class对象把类中的各种成员映射成对应的Java类。一个类中的：成员变量、构造函数、成员方法都有对应的Java类：Field、Contructor、Method。

3.我们可以通过Class类的newInstance()来创建真实对象，当然这个方法需要对象里提供无参构造,否则会出现 java.lang.InstantiationException.

4.sun公司专门给我们提供了如何获取一个类中的所有构造函数。Constructor[] getConstructors(), Constructor[] getDeclaredConstructors(),
Constructor getConstructor(Class … parameterTypes),Constructor getDeclaredConstructor(Class … parameterTypes).获取到构造函数对象后，就可以创造某个真实对象。注意暴力访问时候，需要setAccessible(true);
